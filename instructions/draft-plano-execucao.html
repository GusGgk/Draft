Por que essa separação?

Cada camada pode ser modificada de forma independente. Se trocarmos o banco de MySQL para PostgreSQL no futuro, o React não precisa mudar nada. Se redesenharmos a interface, o banco não é afetado. Isso é chamado de separação de responsabilidades.

O que é Docker — e como usamos

O problema que o Docker resolve

Imagine que o André instala o MySQL no Windows, o Eduardo usa Linux e o João usa Mac. Cada sistema pode ter uma versão diferente do MySQL, configurações diferentes, e um comportamento diferente. Um bug no banco do André pode não aparecer na máquina do Eduardo. Isso gera o famoso problema: "na minha máquina funciona".

O Docker resolve isso empacotando o MySQL em um container — uma mini-máquina virtual padronizada que funciona de forma idêntica em qualquer sistema operacional.

📦
Analogia: Contêiner de navio

Assim como um contêiner de navio tem tamanho e encaixes padronizados (e pode ser transportado por qualquer navio), um container Docker tem uma interface padronizada e roda em qualquer máquina com Docker instalado.

O arquivo docker-compose.yml — linha a linha

O arquivo draft-back-end/docker-compose.yml atual define como o MySQL vai rodar:

# Versão do formato do docker-compose
version: '3.8'

# Lista de serviços (containers) que serão criados
services:
  mysql:                        # Nome do serviço (você escolhe)
    image: mysql:8.0            # Usa a imagem oficial do MySQL 8.0
    container_name: draft-mysql # Nome legível do container
    restart: always              # Reinicia automaticamente se cair
    ports:
      - "3307:3306"             # Sua máquina:porta → Container:porta
                                   # 3306 é a porta padrão do MySQL
                                   # Usamos 3307 na sua máquina para
                                   # não conflitar com MySQL local
    environment:               # Variáveis de ambiente passadas ao MySQL
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: draft_db  # Cria esse banco automaticamente
      MYSQL_USER: draft_user
      MYSQL_PASSWORD: secretpassword
    volumes:
      - mysql-data:/var/lib/mysql # Salva os dados FORA do container
                                   # Sem isso, os dados somem ao reiniciar

volumes:
  mysql-data:                    # Declara o volume para persistência

Comandos essenciais do Docker

ComandoO que faz

O que é MySQL — e como modelamos o banco

O conceito de banco relacional

O MySQL é um banco de dados relacional. Isso significa que os dados são armazenados em tabelas (como planilhas do Excel), e as tabelas se relacionam entre si por meio de chaves.

ConceitoAnalogiaNo Draft
TabelaPlanilha do Excelusers, posts, follows
Linha (registro)Linha da planilhaUm usuário específico
Coluna (campo)Coluna da planilhaname, email, created_at
Chave primária (PK)CPF — identificador únicoid auto-incremental em cada tabela
Chave estrangeira (FK)Referência cruzadaposts.user_id → users.id

O que é uma Migration?

Uma migration é um arquivo PHP que descreve uma alteração no banco de dados. Em vez de criar tabelas manualmente em um cliente SQL, você escreve um arquivo PHP com as instruções. O Laravel executa esses arquivos em ordem com o comando php artisan migrate.

✅
Por que migrations são melhores?

Porque ficam versionadas no git. Toda a equipe vê exatamente o que foi criado, quando foi criado, e pode reproduzir o mesmo banco em qualquer máquina com um único comando.

Anatomia de uma migration:

// Arquivo: database/migrations/2026_03_30_create_posts_table.php

return new class extends Migration
{
    // up() é executado ao rodar: php artisan migrate
    public function up(): void
    {
        Schema::create('posts', function (Blueprint $table) {
            $table->id();               // Coluna id, PK auto-incremental
            $table->foreignId('user_id')
                  ->constrained('users')  // FK para a tabela users
                  ->cascadeOnDelete();        // Deleta posts se o user for deletado
            $table->text('body');          // Texto do post
            $table->timestamps();           // created_at e updated_at automáticos
        });
    }

    // down() é executado ao rodar: php artisan migrate:rollback
    public function down(): void
    {
        Schema::dropIfExists('posts');
    }
};

O que é Laravel — e como se conecta ao banco

O padrão MVC

Laravel organiza o código no padrão MVC (Model, View, Controller). No contexto de uma API, funciona assim:

CamadaResponsabilidadeExemplo no Draft
ModelRepresenta uma tabela do banco. Sabe como ler e gravar dados.Post.php, User.php
ControllerRecebe a requisição HTTP, usa o Model e retorna uma resposta.PostController.php
ViewEm APIs, é substituída por JSON. O React é quem "renderiza a view".Resposta JSON para o React

O que é Eloquent ORM?

Eloquent é o sistema do Laravel que permite trabalhar com o banco de dados usando PHP puro, sem escrever SQL manualmente. Cada Model corresponde a uma tabela.

// Sem Eloquent — SQL puro (difícil de manter):
$posts = DB::select('SELECT * FROM posts WHERE user_id = ? ORDER BY created_at DESC', [$userId]);

// Com Eloquent — muito mais legível:
$posts = Post::where('user_id', $userId)->latest()->get();

// Relacionamentos — busca posts COM os dados do autor:
$posts = Post::with('user')->latest()->paginate(20);

Banco Centralizado — o Problema e a Solução

Por que o Docker local não é suficiente

O docker-compose.yml atual cria um container MySQL que roda na máquina de cada pessoa. Isso significa:

SituaçãoDocker localBanco centralizado
André cria um usuário de testeAparece só na máquina do AndréAparece para todos
Eduardo roda as migrationsAltera só o banco deleAltera o banco compartilhado
Gustavo quer ver os dadosPrecisa criar os mesmos dados de novoJá estão lá
Banco fica offlineQuando a máquina é desligadaNunca (servidor na nuvem)
🚫
O problema real

Se cada membro tiver seu banco local, vocês estarão desenvolvendo em isolamento. Inconsistências de dados entre membros vão causar bugs difíceis de reproduzir.

Solução recomendada: Railway.app

Railway é uma plataforma de hospedagem que oferece MySQL 8.0 gratuito para projetos de desenvolvimento. Em 5 minutos, a equipe toda está conectada ao mesmo banco.

✅
Por que Railway?

Gratuito • MySQL 8.0 idêntico ao docker-compose.yml • Sempre online • Configuração em 5 minutos • Sem cartão de crédito

Passo a passo — Configurar o Railway (uma vez pelo Tech Lead)

  1. Criar conta no Railway
    Acesse railway.app e faça login com sua conta do GitHub.
  2. Criar o banco MySQL
    Clique em "New Project" → "Provision MySQL". Aguarde ~30 segundos.
  3. Copiar as credenciais
    Clique no serviço MySQL → aba "Variables". Copie: MYSQLHOST, MYSQLPORT, MYSQLDATABASE, MYSQLUSER, MYSQLPASSWORD.
  4. Compartilhar com a equipe
    Envie as credenciais via Discord privado. NUNCA coloque no git.
  5. Configurar o .env e rodar as migrations
    Preencha o .env com as credenciais do Railway e execute php artisan migrate. Só precisa ser feito uma vez.

Integração Laravel + React

O que é uma API REST?

Uma API REST é um contrato de comunicação entre o frontend (React) e o backend (Laravel). O React faz requisições HTTP para URLs específicas, e o Laravel responde com JSON.

Verbo HTTPUsoExemplo no Draft
GETBuscar dadosGET /api/posts — lista posts do feed
POSTCriar dadosPOST /api/posts — publica um novo post
PUT/PATCHAtualizar dadosPATCH /api/profile — edita o perfil
DELETEDeletar dadosDELETE /api/posts/42 — deleta o post 42

Laravel Sanctum — Autenticação da API

O Sanctum é o sistema de autenticação do Laravel para APIs. Quando o usuário faz login, o Laravel gera um token. O React salva esse token e o envia em todas as requisições seguintes no cabeçalho Authorization.

No Laravel — definindo as rotas (routes/api.php):

// Rotas públicas (sem autenticação)
Route::post('/register', [AuthController::class, 'register']);
Route::post('/login',    [AuthController::class, 'login']);

// Rotas protegidas (exigem token)
Route::middleware('auth:sanctum')->group(function () {
    Route::get('/posts',   [PostController::class,    'index']);
    Route::post('/posts',  [PostController::class,    'store']);
    Route::get('/profile', [ProfileController::class, 'show']);
});

No Laravel — o Controller:

class PostController extends Controller
{
    // Chamado quando React faz GET /api/posts
    public function index()
    {
        $posts = Post::with('user')   // inclui dados do autor
                        ->latest()      // mais recentes primeiro
                        ->paginate(20); // 20 por página

        return response()->json($posts); // retorna JSON para o React
    }
}

No React — fazendo a requisição com Axios:

// src/hooks/usePosts.js
import axios from 'axios';
import { useState, useEffect } from 'react';

// Configura a URL base e o token de autenticação
axios.defaults.baseURL = 'http://localhost:8000/api';
axios.defaults.headers.common['Authorization'] =
  `Bearer ${localStorage.getItem('token')}`;

export function usePosts() {
  const [posts, setPosts] = useState([]);

  useEffect(() => {
    axios.get('/posts')
      .then(res => setPosts(res.data.data))
      .catch(err => console.error(err));
  }, []);

  return posts;
}
⚠️
CORS — comunicação entre portas diferentes

O React roda na porta 5173 e o Laravel na porta 8000. O browser bloqueia isso por padrão. Para liberar, adicione no .env do Laravel: SANCTUM_STATEFUL_DOMAINS=localhost:5173 e no config/cors.php adicione a origem do frontend.

Glossário

Migration

Arquivo PHP que descreve uma alteração no banco de dados. Rodadas em ordem pelo comando php artisan migrate. Funcionam como "controle de versão do banco de dados".

Seeder

Arquivo PHP que insere dados iniciais ou de teste no banco. Rodado com php artisan db:seed. Útil para criar dados padrão como lista de esportes ou usuário admin inicial.

Model (Eloquent)

Classe PHP que representa uma tabela do banco. Permite ler e gravar dados sem escrever SQL. Ex: Post::all() retorna todos os posts da tabela posts.

Controller

Classe PHP que recebe requisições HTTP, executa a lógica de negócio usando Models, e retorna uma resposta JSON. É o intermediário entre as rotas e o banco.

Chave Primária (PK)

Coluna que identifica unicamente cada registro de uma tabela. Geralmente chamada id e é auto-incremental (1, 2, 3...). É como o CPF de cada linha.

Chave Estrangeira (FK)

Coluna que referencia a chave primária de outra tabela. Ex: posts.user_id referencia users.id. O banco garante que não exista um post apontando para um usuário inexistente.

Soft Delete

Técnica de "deletar sem deletar". Em vez de remover o registro, uma coluna deleted_at recebe a data/hora. O Laravel automaticamente ignora esses registros nas consultas, mas permite recuperá-los.

Relação polimórfica

Quando uma tabela pode se relacionar com múltiplas outras tabelas. Ex: likes pode pertencer tanto a um Post quanto a um Comment. Implementado com colunas _type e _id.

Sanctum

Pacote oficial do Laravel para autenticação de APIs via tokens. Quando o usuário faz login, recebe um token. Esse token é enviado em cada requisição para provar sua identidade.

Artisan

Ferramenta de linha de comando do Laravel. Chamada via php artisan <comando>. Permite criar arquivos, rodar migrations, seeds, iniciar o servidor e muito mais.

CORS

Mecanismo de segurança do browser que bloqueia requisições entre domínios/portas diferentes. Como React (5173) e Laravel (8000) estão em portas diferentes, o Laravel precisa explicitamente permitir as requisições do React.

Container Docker

Uma unidade de software isolada que empacota uma aplicação e todas as suas dependências. Roda de forma idêntica em qualquer máquina. No Draft, usamos um container para o MySQL.

Volume Docker

Mecanismo para persistir dados gerados por um container. Sem um volume, todos os dados do MySQL somem quando o container é parado. O volume salva os dados fora do container.

API REST

Estilo de arquitetura para comunicação entre sistemas via HTTP. Define convenções de URL e verbos (GET, POST, PUT, DELETE). O Laravel expõe endpoints REST que o React consome.

JSON

Formato de troca de dados entre backend e frontend. É texto simples estruturado em pares chave-valor. Ex: {"name":"João","age":22}. Laravel e React falam esse idioma nativo.
